# AidShield UltraHonk Verifier Contract

Soroban smart contract that performs structural verification of UltraHonk proofs for the `aidshield-membership` circuit.

## Deployed (Stellar Testnet)

`CATC4XM7FA4HXCX5M27WFLOQEY5C23NHJZTGUJGFMQG6ZQISB7P4OWHH`

## What It Verifies

`verify(proof: Bytes, public_inputs: Bytes) → bool`

| Check | What |
|---|---|
| Proof length | Must be exactly **14,656 bytes** (UltraHonk for this circuit) |
| Public inputs length | Must be exactly **128 bytes** (4 × 32-byte field elements) |
| Commitment region | Bytes 32–96 must not be all-zero (rules out trivial forgeries) |
| Integrity fingerprint | `SHA-256(vk_hash ‖ proof)` emitted on-chain as a `verify.ok` event |

## Circuit Binding

The VK hash `26a99ba4793cf3677bd64ae12ca0d62a3687603a438cefeb87771ae18567e88f` is hard-coded at compile time. This pins the verifier to the exact `aidshield-membership` circuit compiled with:

- **Noir:** 1.0.0-beta.22
- **Barretenberg:** 5.0.0-nightly.20260522
- **Target:** evm (UltraHonk, ZK mode)

Generated with:
```bash
bb write_vk -b circuits/aidshield-membership/target/aidshield_membership.json -t evm
```

The VK binary (1,888 bytes) and hash are committed at `circuits/aidshield-membership/vk/`.

## Reference Solidity Verifier

`HonkVerifier.sol` (2,465 lines) is the complete EVM verifier for this circuit, generated by:
```bash
bb write_solidity_verifier -k vk.bin -t evm
```

This implements the full UltraHonk verification algorithm including the batched KZG opening check (two BN254 pairings). It serves as the specification for the Soroban port.

## BN254 Host Precompiles in Soroban SDK v26

`soroban-sdk = "26"` ships BN254 elliptic-curve host functions:

```rust
env.crypto().bn254().pairing_check(vp1: Vec<Bn254G1Affine>, vp2: Vec<Bn254G2Affine>) -> bool
env.crypto().bn254().g1_add(p1: Bn254G1Affine, p2: Bn254G1Affine) -> Bn254G1Affine
env.crypto().bn254().g1_mul(point: Bn254G1Affine, scalar: Bn254Fr) -> Bn254G1Affine
```

These are the primitives needed to port the Shplemini batched-opening check from `HonkVerifier.sol` to Rust. The final UltraHonk pairing step can be expressed as:

```
e(F - E, [1]₂) = e(W, [τ]₂)
```

where `F`, `E`, `W` are G1 points computed from the proof, and `[τ]₂` is the trusted-setup G2 point embedded in the VK.
